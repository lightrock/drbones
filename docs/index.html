Doctor Bones / repo-native cognitionkitecture

Start a project with a repo that can teach the AI how to behave.

Doctor Bones is a repository template for AI-assisted development. The idea is simple: when the repo contains durable intent, workorder rules, checks, examples, and architecture patterns, a foreground AI can understand what kind of work belongs here before an executor touches files.

Open the Miller / PFEM pattern-fit page View the GitHub repository

Simple prompts, smarter repo

If your repo carries cognitionkitecture, you can ask for useful work in plain language. The repo guidance narrows the lane, names the constraints, and tells the AI what evidence or checks matter.

Patterns without vendor lock-in

Doctor Bones can reference generic architecture patterns such as PFEM and PFCOMM without becoming tied to a single AI vendor, IDE, MCP server, or coding-agent product.

Foreground before executor

The foreground AI frames the work. The executor changes files only when the task is bounded, the target repository is correct, and the checks are named.

Example prompt shape

Instead of hand-feeding every instruction, point the foreground AI at the Doctor Bones-based repository and ask for the outcome. The repo should teach the AI the discipline. 

You are the foreground AI for <your project repository URL>.
Current repo state beats chat memory.
Read README.md, AGENTS.md, examples/README.md, and relevant repo guidance first.

Task:
Use the repo's architecture patterns and workorder discipline to help me plan the next safe change.
